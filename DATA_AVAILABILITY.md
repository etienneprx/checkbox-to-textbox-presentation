# Data Availability Statement

## Overview

This document explains the data availability and ethical compliance for the "From Checkbox to Textbox" research project.

---

## Raw Survey Data

### Status: NOT PUBLICLY AVAILABLE

The raw data files (`.sav` format) referenced in this repository have been **removed** and are **not publicly available** for the following reasons:

1. **Ethics Committee Requirements**: The research ethics committee that approved this study requires that raw survey data containing potentially identifiable information be kept confidential and not publicly shared.

2. **Participant Privacy**: The raw data contains:
   - Free-text responses that could potentially identify individuals
   - Demographic combinations that could reduce anonymity
   - Sensitive political and social attitudes

### Missing Raw Data Files

The following data file is referenced in the code but **not included** in this repository:

```
data/datagotchi_choix_ouverts_June 23, 2025_11.19.sav
```

**Impact on Reproducibility**:
- The initial data cleaning pipeline (`R/cleaning__master.R`, `R/cleaning__master_open.R`, `R/cleaning__master_closed.R`) requires this file
- The AI processing pipeline (`R/ai_process_responses.R`) require this file

## Reproducibility Without Raw Data

### What You CAN Reproduce

Even without the raw `.sav` files, researchers can:

1. **Examine the AI processing methodology**:
   - Review the prompting strategy in `docs/prompts/`
   - Understand the consensus mechanism in `R/ai_process_responses.R`
   - Analyze metadata about processing quality

2. **Reproduce all analyses**:
   - All `R/fig_*.R` scripts work with the processed JSON data
   - Factor analyses, matching, regressions can all be verified
   - Visualization code can be adapted to other datasets

3. **Apply the methodology to new data**:
   - The entire pipeline is generalizable
   - `run_all_open_cleaning_prompts.sh` can process new surveys
   - Prompt templates in `docs/prompts/` can be adapted

4. **Validate the approach**:
   - Comprehensive metadata allows methodological assessment
   - Quality metrics are fully documented
   - Code is transparent and well-commented

### What You CANNOT Reproduce

Without the raw data, you cannot:

1. Re-run the initial SPSS data extraction (`extract_questions.R`)
2. Regenerate the AI-cleaned codes from scratch (but all existing codes and metadata are available for verification)
3. Access the original closed-ended responses that served as ground truth

## Transparency and Open Science

While we cannot share raw data due to ethical constraints, we are committed to maximum transparency through:

1. **Open source code**: All analysis code is publicly available
2. **Comprehensive documentation**: Methods are fully documented
3. **Metadata sharing**: Complete processing metadata is included
4. **Methodological transparency**: All prompts, parameters, and procedures are disclosed
5. **Responsive to inquiries**: We welcome questions and will provide additional documentation as needed
